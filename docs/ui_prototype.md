## 十二、关键交互流程：HIL 决策

```mermaid
sequenceDiagram
    participant U as 用户
    participant UI as Web UI / CLI
    participant O as Orchestrator
    participant CP as Checkpoint

    O->>CP: 1. 进入 HIL 前保存 checkpoint
    O->>UI: 2. 推送待审 artifact + Agent 备注
    UI->>U: 3. 显示审阅界面（多 Tab + 决策按钮）
    Note over U: 用户审阅 / 评论

    alt 确认 (y)
        U->>UI: 点击 [✅ 确认]
        UI->>O: decision=confirm
        O->>CP: 标记节点完成
        O->>O: 进入下一节点
    else 修改 (e)
        U->>UI: 编辑 artifact + 点击 [✏️ 修改]
        UI->>O: decision=edit, payload=新内容
        O->>CP: 更新 state
        O->>O: 重新进入当前节点（用新输入）
    else 回退 (r)
        U->>UI: 选择回退目标 + 点击 [⏪ 回退]
        UI->>O: decision=rollback, target=历史节点
        O->>CP: 加载历史 checkpoint
        O->>O: 跳回目标节点
    else 中止 (n)
        U->>UI: 点击 [🛑 中止]
        UI->>O: decision=abort
        O->>CP: 标记项目 aborted
    end
```

## 十三、设计规范

| 项 | 规范 |
|---|---|
| 主色 | `#2563eb`（蓝，科研专业感） |

| 暗色模式 | P2 |
| 国际化 | 中/英，按 config.language 自动切换 |
